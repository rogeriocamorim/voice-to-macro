# AGENTS.md

## What this is

AI-powered voice command system for PC games. Captures mic audio, runs local STT (Whisper) and LLM (Ollama) to classify intent, then dispatches keyboard/mouse actions. Fully local, no cloud.

For Elite Dangerous: includes game state awareness (journal/status/binds), galaxy map UI automation, EDSM/Spansh API integration for trade/profitability/system info, all using the player's real keybindings from their `.binds` XML file.

## Commands

| Task | Command |
|------|---------|
| Install deps | `pip install -r requirements.txt` |
| First-run setup | `python setup.py` |
| Run CLI | `python main.py` |
| Run GUI | `python gui_app.py` |
| Run with profile | `python main.py --profile elite_dangerous` |
| Tests | `pytest` |

No linter, formatter, or CI configured.

## Architecture

Pipeline: Mic → VAD/PTT → Whisper STT → learned-command fast-path → fuzzy match → LLM classification → action dispatch → TTS confirmation.

For compound actions (Elite Dangerous): LLM returns JSON `{"action": "name", "params": {...}}` → handler dispatches multi-step logic (API calls, galaxy map automation, TTS response).

Two entry points:
- `main.py` — CLI event loop
- `gui_app.py` — PyQt6 GUI

Packages map 1:1 to pipeline stages:

| Package | Role |
|---------|------|
| `stt/` | faster-whisper speech-to-text |
| `vad/` | Silero VAD + push-to-talk recorder |
| `agent/` | LLM intent parsing (Ollama), context building, clarification feedback |
| `executor/` | pyautogui key/combo/hold/sequence dispatch + bind-aware dispatch |
| `tts/` | edge-tts (neural, needs internet) with pyttsx3 offline fallback |
| `learning/` | Self-learning command store (`learned_commands.json`, gitignored) |
| `gui/` | PyQt6 GUI components |
| `profiles/` | Game profile JSON files defining available actions |
| `gameapi/` | Elite Dangerous game state: journal watcher, status reader, .binds parser |
| `search/` | External API clients: EDSM (star map data), Spansh (trade routes) |
| `handlers/` | Compound action handlers for Elite Dangerous (navigate, trade, system info) |

## Profiles

JSON files in `profiles/`. Each profile has two sections:
- `actions`: Simple keystroke actions with `type` (`key`, `combo`, `hold`, `sequence`)
- `compound_actions`: Complex multi-step actions handled by Python code (handlers/)

The active profile is set in `config.yaml`. When Elite Dangerous is active, the executor reads the player's `.binds` file for real keybindings, falling back to profile-defined keys.

## Prerequisites

- **Ollama** must be installed and running (`ollama serve`) with the model specified in `config.yaml` pulled.
- **CUDA** recommended for Whisper large-v3. CPU works but is slow; use `small` or `medium` whisper model on CPU.
- **Target OS**: Windows 10/11 (pyautogui, pynput, nvidia-smi, wmic). Development may happen on macOS but runtime assumptions are Windows.
- **httpx** and **watchdog** required for Elite Dangerous integration (journal file watching, API calls).

## Testing

- pytest with `unittest.mock` to mock `pyautogui` and external dependencies.
- Tests live in `tests/` mirroring package structure.
- No integration tests requiring Ollama or a mic — all external I/O is mocked.
- Key test files: `test_binds_parser.py`, `test_journal_watcher.py`, `test_game_state.py`, `test_status_reader.py`, `test_edsm.py`, `test_handlers.py`

## Config

`config.yaml` is the single runtime config file (generated by `setup.py`). Key settings:
- Core: `mode`, `ptt_key`, `active_profile`, `model`, `device`, `whisper_model`, `confidence_threshold`
- Elite: `elite_journal_path`, `elite_binds_file`, `edsm_commander_name`, `edsm_api_key`
- Trade: `trade_max_cargo`, `trade_max_hops`, `trade_max_hop_distance`, `trade_requires_large_pad`, `trade_starting_capital`
